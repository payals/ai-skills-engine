## Usage Patterns

### Before ANY Response
```
1. Parse user message for keywords
2. Match to skill category above
3. Read SKILL.md: `.cursor/skills/{category}/{skill}/SKILL.md`
4. State: "Using {skill name} for {purpose}"
5. Follow skill instructions
```

### Process Skills First
When multiple skills apply:
1. **Process skills** (brainstorming, when-stuck) determine HOW to approach
2. **Implementation skills** (senior-backend, docker-expert) guide execution

### Task Flow Example
```
User: "Add user authentication"
→ Check: brainstorming (design first)
→ Check: senior-backend (API patterns)
→ Check: senior-security (security practices)
→ Check: test-driven-development (TDD)
→ Check: verification-before-completion (before claiming done)
```
